# TS-Go-Accepted Baseline Overlays

TSTS is a mechanical 1:1 port of the pinned TS-Go compiler (`packages/tsts/_vendor/typescript-go`).
The exact-baseline gate of the tsgo-suite compares TSTS output against the Strada-generated
reference baselines vendored under `_submodules/TypeScript/tests/baselines/reference`.

For a small number of artifacts, the **pinned TS-Go compiler itself demonstrably diverges from
the Strada baselines** (chiefly isolated-declarations emit and source-map spans — surface Strada
exercises through its transpile suite but TS-Go never runs). Forcing TSTS to match Strada there
would mean deliberately diverging from TS-Go, breaking the mechanical-port invariant. This is the
same class of drift Microsoft tracks upstream with their `submoduleAccepted`/`submoduleTriaged`
baseline trees for the suites TS-Go does run.

The files in this directory capture **pinned TS-Go's actual output** for exactly the divergent
sections. The runner (`run.mjs`) compares those sections against the overlay and everything else
against the Strada baseline, and reports every overlay use in the run summary
(`Exact-baseline tsgo-accepted sections`).

## Rules

- Overlays are generated by `capture-tsgo-accepted.mjs` from a real pinned-TS-Go binary run.
  Never hand-edit an overlay. Never derive an overlay from TSTS output.
- An overlay section must differ from the corresponding Strada section; the capture tool and the
  runner both fail on stale (identical) overlays.
- Overlay sections may only override sections that exist in the Strada baseline.
- Every overlay file/section pair is declared in `manifest.json`.

## Regenerating

```sh
cd packages/tsts/_vendor/typescript-go
go build -o /tmp/tsgo ./cmd/tsgo
cd ../../../..
node packages/tsts/tools/tsgo-suite/capture-tsgo-accepted.mjs --tsgo /tmp/tsgo
```

Regeneration is required whenever the pinned TS-Go revision changes.
